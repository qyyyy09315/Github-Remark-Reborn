# GithubRemark 重制版
> 本项目是基于原 [GithubRemark]() 插件的适配重制版，针对2025年后新版GitHub页面结构和Turbo框架做了全量适配，解决了原插件失效问题，保留了所有原有功能并优化了性能。

## ✨ 功能特性
- 🔖 给任意GitHub用户添加自定义备注名，类似微信备注功能
- 📝 双击备注即可快速修改，操作便捷
- 📱 全场景覆盖：个人主页、动态首页、关注/粉丝列表、项目星标/关注者列表、组织成员页
- 🚀 高性能：防抖优化，不影响GitHub原生页面加载速度

## 🚀 安装方法
1. 下载本项目所有代码到本地任意目录
2. 打开Chrome浏览器，进入扩展管理页面 `chrome://extensions/`
3. 开启右上角「开发者模式」
4. 点击左上角「加载已解压的扩展程序」，选择刚才下载的项目目录
5. 扩展安装完成，刷新GitHub页面即可使用

chrome插件商店的版本会在以后发布。

## 📖 使用说明
1. 点击Chrome工具栏中的GithubRemark扩展图标，选择`on`开启功能
2. 开启后，所有GitHub用户名后面会显示备注，没有设置过的会显示`(no remark)`
3. 直接双击备注文字，在弹出的输入框中输入新备注，确认即可完成修改
4. 备注会自动同步到云端，更换设备登录同一GitHub账号即可看到所有备注
5. 注意：需要登录GitHub账号才能使用，系统以你的GitHub用户名作为唯一身份标识，保障数据安全

## 🏗️ 技术架构
### 前端扩展（Manifest V3）
- 后台Service Worker：监听页面加载，自动注入功能脚本，管理扩展开关状态
- 内容脚本：`ghremark.js` 核心功能实现，适配新版GitHub页面结构和Turbo单页导航
- 接口层：`webapi.js` 封装云端接口调用，处理异常降级和参数校验
- 支持多端：同时提供Chrome扩展版和油猴脚本版（`tampermonkey`目录下）

### 后端服务（可选自行部署）
- 基于Node.js实现的轻量HTTP服务，源码在`server`目录下
- 提供两个核心接口：`getRemark` 查询备注、`updateRemark` 更新备注
- 默认使用公共服务接口，也可以自行部署替换`webapi.js`中的服务地址
